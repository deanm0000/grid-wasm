# grid-wasm — Agent Reference

A high-performance data grid built on Rust + WebAssembly + React.
Rust owns all rendering, hit-testing, state, and DataFusion queries.
JS/TS owns the DOM, event wiring, HTML overlays (menus, scrollbars, color pickers), and the public API.

## Build commands

| Command | What it does |
|---------|-------------|
| `npm run wasm` | Compile Rust → WASM (dev, no wasm-opt, ~25 s). Always run after any `.rs` change. |
| `npm run wasm:release` | Compile Rust → WASM with full optimisation (~10-30 min). For production only. |
| `npm run build` | TypeScript type-check + Vite bundle. Run after any `.ts`/`.tsx` change. |
| `npm run build:all` | `wasm` + `build` together. |
| `npm run dev` | Vite dev server at http://localhost:5173. Required by Playwright. |
| `npm test` | Playwright tests (auto-starts Vite). |
| `npm run lint` | ESLint. |

**After any Rust change:** run `npm run wasm` before `npm run build` or `npm test`.
TypeScript errors about missing WASM methods are expected until `npm run wasm` regenerates `src/wasm/grid_core.d.ts`.

## Architecture principles

### Single source of truth for hit areas
`layout.rs` → `ColumnLayout` / `ColumnLayoutEntry` — computed once per render in `GridState::recompute_layout()`, used by ALL draw and hit-test code. Never compute column positions independently; always read from `col_layout`.

### Coordinate system
All internal coordinates are **WASM pixels** = `CSS pixels × devicePixelRatio`.
Mouse events from JS multiply by DPR before passing to WASM:
```typescript
const x = (e.clientX - rect.left) * dpr;
```
The canvas CSS size always equals `canvas.width / dpr` (set via `canvas.style.width`).

### Canvas ownership
`CanvasCtx` lives on `DataGrid` (in `lib.rs`), **not** on `GridState`.
`GridState::render(&mut CanvasCtx)` receives the context as a parameter.
JS overlays (menus, scrollbars, color pickers) are HTML elements on `document.body` — they never touch the canvas.

### Column coordinates
`MappedColumn.source_index` = display order position.
`MappedColumn.arrow_index` = position in the Arrow schema.
Cell data lookup: `display_index → arrow_index → ArrowDataSource::get_cell(arrow_index, row)`.

### `effective_columns()` contract
Returns only columns whose left edge is before the right canvas boundary (`draw_x < self.width`). Columns partially behind sticky frozen columns are **included** (the `diff` mechanism in renderers clips them). Never skip off-left columns — `walk_columns` needs them for correct x accumulation.

## Key design patterns

### Adding a new WASM binding
1. Add the method to `impl DataGrid` in `lib.rs` with `#[wasm_bindgen]` on the impl block.
2. Run `npm run wasm` — this regenerates `src/wasm/grid_core.d.ts`.
3. Add the corresponding method/prop to `WasmDataGrid` in `grid.ts`.

### Adding a new field to GridState
1. Add to `pub struct GridState` in `grid.rs`.
2. Initialise in `GridState::new()`.
3. If the field affects rendering, pass it through `render()` → `draw_grid()` → relevant renderer.

### Passing data from JS to WASM
Prefer JSON strings for complex objects (parsed with `serde_json::from_str` in Rust) rather than `serde-wasm-bindgen` for structured types. Primitives (`f64`, `bool`, `&str`) cross the boundary directly.

### Conditional formats
Rules are stored in `GridState::conditional_format_overrides: HashMap<String, Vec<ConditionalRule>>` keyed by `arrow_name`. They are evaluated per-cell in `render/cells.rs::evaluate_conditions()` and take priority over `data_style.conditional_formats` from `ColumnInput`.

### Date/datetime columns
`created_at`-style columns are stored as `Utf8` Arrow arrays (ISO 8601 strings from JSON).
Date inference (`infer_date_truncations`) is triggered at data load via `auto_configure_from_data` and caches results in `GridState::date_truncation_cache`.
All code that needs to detect "is this a date-like column?" should check `date_truncation_cache.contains_key(arrow_name)` for Utf8 columns in addition to the native `Date32 | Date64 | Timestamp` Arrow types.

## Testing

Playwright tests in `tests/hit-areas.spec.ts` cover:
- Draw vs layout coordinate agreement (all column positions match between draw and hit-test paths)
- Static geometry invariants (non-overlapping zones, symmetry, y-coverage)
- Live sweep: resize zone straddles column border in both LTR and RTL directions

`window.__gridDebug` is set in `App.tsx` to the internal WASM `DataGrid` instance, giving Playwright direct access to WASM methods.

Run tests: `npm test`
The test server auto-starts via `playwright.config.ts` webServer config.

## Known gotchas

- **`draw_cells` width check**: uses `col_draw_x >= width` (canvas width), not height. The original code had `col_draw_x > height` which caused cells to disappear when x > canvas height.
- **`recompute_layout` must be called before hit-testing**: `on_mouse_down` calls `self.state.recompute_layout()` to ensure the layout reflects the current scroll/column state.
- **`translate_x` and column visibility**: `effective_columns()` uses actual `draw_x = sticky_width + translate_x + cumulative_col_widths` to determine visibility, not just raw accumulated width. This is critical for correct horizontal scrolling.
- **getrandom dual versions**: Both `getrandom 0.2` (needs `js` feature) and `getrandom 0.3` (needs `wasm_js` feature) are required in `Cargo.toml` due to transitive dependencies from DataFusion. Do not consolidate.
- **chrono `wasmbind` feature**: Required in `Cargo.toml` so `Utc::now()` uses `js_sys::Date` instead of `SystemTime::now()` (unsupported on wasm32).
- **`wasm-pack --out-dir` is relative to the crate**: The npm script uses `--out-dir ../../src/wasm` (relative to `crates/grid-core/`), which outputs to `src/wasm/` at the project root.
